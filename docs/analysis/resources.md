# Recursos

Recursos son herramientas, documentos y medios usados para ejecutar los
procesos de Zarvent Repuestos.

## Recursos tecnologicos del proyecto

| Recurso | Uso academico |
| --- | --- |
| MySQL Server | Gestor relacional exigido para implementar el esquema del proyecto. |
| SQL | Lenguaje para crear tablas, claves, restricciones, inserts y consultas. |
| MySQL Workbench / cliente MySQL | Herramienta para ejecutar y revisar el esquema fisico. |
| Visual Studio Code | Editor para documentacion y archivos SQL. |
| Git / GitHub | Control de versiones y respaldo del avance del proyecto. |

## Recursos actuales del negocio

| Recurso | Informacion que contiene |
| --- | --- |
| Cuaderno o libreta de ventas | ventas, productos vendidos, montos y fechas |
| Excel de productos y precios | catalogo, stock aproximado, precios y costos |
| WhatsApp | pedidos, consultas, cotizaciones y acuerdos con clientes o proveedores |
| Facturas y recibos de venta | cliente, monto, fecha, producto y comprobante |
| Facturas de compra | proveedor, repuestos comprados, cantidades y costos |
| Notas de almacen | ingresos, salidas, ubicaciones y ajustes |
| Fotografias o codigos de repuestos | apoyo para identificar piezas similares |
| Formularios de devolucion o garantia | venta original, motivo, resolucion y estado del producto |

## Utilidad para la base de datos

Estos recursos justifican que la informacion se centralice en tablas
relacionadas. La base de datos no reemplaza el analisis del negocio; organiza
lo que hoy esta disperso en papeles, planillas y mensajes.
